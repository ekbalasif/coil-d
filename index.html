About Us

The Mission Bhashini, an initiative by the Ministry of Electronics and Information Technology (MeitY), Government of India, has recently recognized and funded the establishment of the "COIL-D: Centre of Indian Language Data." This esteemed project is being led by Prof. Asif Ekbal, Associate Professor at IIT Patna, in collaboration with IIT Delhi, IIIT Delhi, IIT Guwahati, IGDTUW, MIT Manipal, and DIBD.

Objective

The COIL-D (Centre of Indian Language Data) project aims to establish a unified repository of Indian language data and create a platform for the systematic evaluation of Machine Translation (MT) and other Natural Language Processing (NLP) systems. The project will focus on promoting the creation and preservation of language resources for Human Language Technology (HLT) applications. It will also work on developing standards, guidelines, and best practices for using these resources to benchmark various linguistic tools.

The specific goals of the COIL-D project are:

  • Identification of Hindi language resources
  • Acquisition of resources across target domains
  • Creation of language resources, particularly for Machine Translation, and benchmarks for NLP tools
  • Development of leaderboards for evaluating Machine Translation systems
  • Creation of leaderboards for Automatic Speech Recognition (ASR) and Text-to-Speech (TTS) technologies
  • Establishment of benchmarks for linguistic tools like PoS taggers, NER taggers, etc

Motivation

A significant emphasis is placed on the creation of high- quality language resources. Efforts will be made to generate parallel corpora, aiming for 50k-100k sentence pairs for each language pair, with 2k additional sentence pairs for evaluation purposes. Hindi will serve as a central language, enabling efficient translation between Hindi and other Indian languages. Furthermore, the creation of leaderboards for MT, ASR, and TTS will provide standardized platforms for performance benchmarking. We will also establish benchmarks for various linguistic tools, such as PoS taggers, Named Entity Recognition (NER) systems, Natural Language Generation (NLG), and Sentiment Analysis tools, which will enhance resources available for Indian languages.

Leader Board

A leaderboard is a platform or framework used to benchmark and compare the performance of different models and algorithms on specific natural language processing tasks. In this case, the leaderboard compares the performance of Machine Translation systems for Indian Languages. The users of this system can be classified broadly into admin, developers and viewers. The admin side will maintain the leaderboard, updating it with new benchmarking datsets, adding newer tasks and other features to the existing framework. The developers are users who would want to compare their MT hypothesis again state of the art model hypothesis. Lastly, the viewers are the visitors to the website.

Hypothesis Upload
The developer can choose their desired language pair and its corresponding test set. They can then run their models on the test set and get the model hypothesis, which is then uploaded to the leaderboard.

Model Upload
Developers can also upload their model weights and configurations to the leaderboard to host it on the website. This will enable all viewers to use an inference api and run the models on sample texts in the server itself.

Score Calculation
The uploaded hypothesis is then evaluated by the leaderboard against reference translations based on various MT metrics. The MT system is then ranked alongside the existing systems and displayed accordingly on the leaderboard.

Evaluation metrics
The following evaluation metrics are considered for evaluating the systems:

  • BLEU: It compares the n-gram overlap between reference and hypothesis translation and scores the hypothesis between 0 to 100, where higher scores correspond to greater similarity between the two and indicate better translation.
  • TER: It measures the number of edits (insertions, deletions, substitutions, and shifts) required to transform the system-generated translation into a reference translation, normalized by the length of the reference. Here lower scores correspond to better translation.
  • CHRF:It measures the harmonic mean of precision and recall over character n-grams, capturing finer-grained similarities between translations and references.
  • COMET: It leverages pre-trained language models to compare the embedding similarity between source, reference and hypothesis and maps it to human judgment scores.
  • BERTScore: This metric uses contextual embeddings from pretrained BERT models to calculate the similarity between the predicted translation and reference translations at the token level.

The Model upload feature comes with the following functionalities:
  • Model upload page : Model weights, requirements and model inferencing python file upload
  • Create Environment page : Create a conda environment with all the dependencies installed on the server.
  • Model inferencing page : Create an inferencing page to select the model and do the inferencing using selected model.
